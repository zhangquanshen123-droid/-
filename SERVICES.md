# Services

This repository documents legal, defensive, and reviewable security-adjacent development services.

Start here: [Quick start for buyers](docs/quick-start-for-buyers.md)

## Available Work

### Repository Setup

- README cleanup
- SECURITY.md policy setup
- Pull request and issue templates
- `.gitignore` setup
- Basic contributor workflow documentation

See [service packages](docs/service-packages.md) for suggested fixed-scope offers.

### Defensive Code Review

- Small-scope source review
- Bug and regression risk review
- Test gap review
- Dependency and configuration review
- Safe remediation notes

### Fix Implementation

- Focused bug fixes
- Test additions
- Documentation updates
- GitHub Actions troubleshooting
- Pull request preparation

## Boundaries

- No unauthorized testing.
- No exploit development.
- No intrusive scanning.
- No credential handling.
- No automated payout or payment account operations.
- Work starts only after scope and authorization are clear.

## Preferred Inputs

- GitHub repository link
- Issue or pull request link
- Scope or program policy link
- Expected deliverable
- Deadline and budget

## Output

Each task should produce one or more of:

- Pull request
- Patch
- Test results
- Review notes
- Safe audit summary
- Cost and reward ledger entry
