## Fixed-Scope Offers

### Starter Repository Setup

- README structure cleanup
- SECURITY.md setup
- Issue template
- Pull request template
- `.gitignore` review

Suggested scope: one small repository.

### Defensive Review Lite

- Review one small module or pull request
- Identify bugs, test gaps, and safe hardening opportunities
- Provide review notes or a patch

Suggested scope: one issue, one PR, or up to a small file set.

### GitHub Actions Fix

- Inspect one failing workflow
- Propose or implement a focused fix
- Document validation steps

Suggested scope: one failing workflow.

## What To Ask Before Starting

- Repository link
- Proof of authorization
- Desired deliverable
- Deadline
- Budget or reward range
- Whether public PRs are acceptable

## Red Lines

- No unauthorized testing.
- No spam.
- No credential handling.
- No exploit payload delivery.
- No payment-account operation.
- No private data extraction.
