# Service Packages

These packages are designed for legal, authorized, defensive work with clear deliverables.

Prices are placeholders. Confirm budget and payment terms before starting.

## Starter Repository Setup

Best for new or messy GitHub repositories that need a clean collaboration baseline.

### Deliverables

- README structure cleanup
- SECURITY.md setup
- Pull request template
- Issue template
- Basic `.gitignore` review
- Short handoff note

### Inputs Needed

- Repository link
- Authorization confirmation
- Preferred project description
- Any existing contribution rules

### Suggested Price Range

- USD 25-75 for a small repository

## Defensive Review Lite

Best for a small module, pull request, or focused file set.

### Deliverables

- Review notes
- Bug and test-gap summary
- Safe hardening suggestions
- Optional patch when the fix is clear

### Inputs Needed

- Repository, issue, or pull request link
- Exact files or module to review
- Authorization confirmation
- Expected output format

### Suggested Price Range

- USD 50-150 for a small, focused review

## GitHub Actions Fix

Best for one failing workflow or CI job.

### Deliverables

- Root-cause summary
- Workflow patch or configuration fix
- Validation notes
- Pull request when authorized

### Inputs Needed

- Repository link
- Failing workflow run or log
- Permission to open a branch or pull request

### Suggested Price Range

- USD 40-120 for one workflow issue

## Security Documentation Pack

Best for projects that need clear disclosure and triage process without active testing.

### Deliverables

- SECURITY.md
- Vulnerability intake issue template
- Disclosure handling notes
- Maintainer checklist

### Inputs Needed

- Repository link
- Preferred contact process
- Public/private disclosure preference

### Suggested Price Range

- USD 25-100 depending on detail

## Start Criteria

Do not start until all are clear:

- Authorized scope
- Deliverable
- Deadline
- Budget or reward range
- Public vs private handling

## Stop Criteria

Stop and ask before continuing when:

- Scope is unclear
- Work touches production users or third-party accounts
- Credentials or private data appear
- Cost is likely to exceed 50% of estimated reward
- The requested work becomes exploitative or intrusive
