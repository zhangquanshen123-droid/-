# Security Policy

## Supported Scope

This repository is available for defensive code review and security fixes only after source code and explicit scope are added.

## Reporting

Do not publish sensitive vulnerability details in public issues unless the project owner has approved public disclosure.

For now, use private project-owner communication for security-sensitive findings.

## Rules For Contributors

- Keep testing non-invasive and limited to authorized code.
- Do not run intrusive scans against third-party systems.
- Do not include secrets, tokens, credentials, or personal data in issues, commits, logs, or pull requests.
- Include clear reproduction notes only when they are safe, minimal, and authorized.
- Prefer fixes, tests, and documentation over exploit proof-of-concept material.
